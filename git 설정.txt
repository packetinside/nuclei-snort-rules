1. 해당 폴더로 이동 (이미 해당 위치라면 생략)
cd C:\projects\packetinside.github.io

2. Git 저장소 초기화

git init
이 명령을 입력하면 폴더 내에 .git이라는 숨겨진 폴더가 생성됩니다.

3. 원격 저장소(GitHub) 연결
이미 생성해둔 GitHub 리포지토리 주소가 있다면 연결합니다.

git remote add origin https://github.com/packetinside/packetinside.github.io.git

4. 브랜치 이름 설정 ('main'으로 통일)
코드 상단 설정에 GIT_BRANCH = "main"이라고 되어 있으므로, 로컬 브랜치 이름도 main으로 맞춰야 오류가 없습니다.

git branch -M main

5. 최초 1회 수동 업로드 (권장)
스크립트가 자동으로 실행되기 전에, 인증 문제나 충돌 방지를 위해 수동으로 한 번 업로드해보는 것이 좋습니다.

git add .
git commit -m "Initial commit"
git push -u origin main

💡 참고 사항
위 과정을 완료한 후 스크립트를 다시 실행하면, git_push_changes 함수가 정상적으로 작동하여 변경된 파일들을 GitHub에 자동으로 푸시할 것입니다.
만약 git push 과정에서 로그인 창이 뜨거나 인증 오류가 난다면, GitHub 계정의 Personal Access Token을 사용하여 인증을 완료해 두어야 스크립트가 백그라운드에서 실행될 때 멈추지 않습니다.
